Purpose

This page focuses only on authentication and token lifecycle behavior: register, login, refresh, and logout.

Register Flow

  1. POST /api/v1/auth/register enters AuthController.register.
  2. AuthService.register checks duplicate email.
  3. Password is encoded with BCrypt.
  4. User is persisted in DB.
  5. Access and refresh JWT tokens are generated.

Login Flow

  1. POST /api/v1/auth/login enters AuthController.login.
  2. LoginAttemptService.assertLoginAllowed enforces lockout policy.
  3. Email + password are validated.
  4. On failure: login-attempt counters are incremented.
  5. On success: counters are cleared and new tokens are returned.

Refresh Flow (Token Rotation)

  1. POST /api/v1/auth/refresh checks that token type is refresh.
  2. Revocation store is checked.
  3. User is loaded by token subject (email).
  4. New access + refresh tokens are generated.
  5. Old refresh token is revoked.

Logout Flow

  1. POST /api/v1/auth/logout validates refresh token type.
  2. Refresh token is added to revocation store.
  3. API returns Logged out successfully.

JWT Claims Used

  • sub: user email
  • userId: user id
  • role: EMPLOYEE, MANAGER, ADMIN
  • type: access or refresh

Quick Curl Sequence

# Register
curl -i -X POST http://127.0.0.1:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","name":"User","password":"StrongPass123!"}'

# Login
curl -i -X POST http://127.0.0.1:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"StrongPass123!"}'
