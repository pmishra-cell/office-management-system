1) Current Scope

  • Single Spring Boot auth service with JWT-based security
  • Register, login, refresh, logout flows
  • Role-based access checks on secure endpoints
  • Brute-force lockout and refresh token revocation
  • Optional Redis-backed state for revocation/lockout

3) End-to-End Flows

3.1 Register Flow

  1. POST /api/v1/auth/register enters AuthController.register.
  2. AuthService.register checks duplicate email in UserAccountRepository.existsByEmail.
  3. Password is BCrypt-encoded.
  4. User is saved to DB (users table).
  5. JwtTokenService generates access + refresh token.
  6. Returns AuthResponse with user metadata and tokens.

3.2 Login + Lockout Flow

  1. POST /api/v1/auth/login enters AuthController.login.
  2. LoginAttemptService.assertLoginAllowed checks if identity is currently locked.
  3. AuthService.login validates password.
  4. On failure: registerFailedAttempt updates counters and lock state.
  5. On success: registerSuccessfulLogin clears attempt state.
  6. Returns fresh access + refresh token pair.

3.3 Refresh Flow (Rotation + Revocation)

  1. POST /api/v1/auth/refresh enters AuthController.refresh.
  2. AuthService.refresh validates token type must be refresh.
  3. Checks revocation store (isRefreshTokenRevoked).
  4. Loads user by email from token claims.
  5. Generates new access + refresh token.
  6. Revokes old refresh token (token rotation).

3.4 Logout Flow

  1. POST /api/v1/auth/logout validates refresh token type.
  2. Refresh token is revoked using the active revocation store.
  3. API responds with message Logged out successfully.

3.5 Protected Endpoint Authorization Flow

  1. Request enters JwtAuthenticationFilter.
  2. Filter reads Authorization: Bearer <token>.
  3. Token is parsed, must have claim type=access.
  4. Spring SecurityContext is populated with principal email + role authority.
  5. @PreAuthorize checks in SecureAccessController enforce role rules.

4) Store Selection: In-Memory vs Redis

Store beans are selected by security.redis.enabled.

  • false (default):
    • InMemoryRefreshTokenRevocationStore
    • InMemoryLoginAttemptStore
  • true:
    • RedisRefreshTokenRevocationStore
    • RedisLoginAttemptStore
Redis mode supports shared security state across multiple service instances.

5) Configuration and Runtime Profiles

Main Config

  • src/main/resources/application.yml
  • Defines H2, JWT expiry, login lockout policy, default security.redis.enabled=false.

Local Redis Profile

  • src/main/resources/application-local-redis.yml
  • Enables Redis mode and points to 127.0.0.1:6379.

Redis Infrastructure

  • docs/docker-compose.redis.yml starts local Redis container with health check.

6) Local Run Scripts

Script What It Does
docs/run-local-redis.sh Starts Redis via compose and runs app with local-redis profile. Handles signal exits (130/143) as graceful termination.
docs/stop-local-redis.sh Stops and removes local Redis compose resources.

8) File-by-File Map (Main Code)

Application Entry

  • src/main/java/com/smartoffice/SmartOfficeApplication.java: Spring Boot entry point.

Auth API DTOs and Controller

  • src/main/java/com/smartoffice/auth/api/AuthController.java: register/login/refresh/logout endpoints.
  • src/main/java/com/smartoffice/auth/api/RegisterRequest.java: register input model.
  • src/main/java/com/smartoffice/auth/api/LoginRequest.java: login input model.
  • src/main/java/com/smartoffice/auth/api/RefreshTokenRequest.java: refresh/logout input model.
  • src/main/java/com/smartoffice/auth/api/AuthResponse.java: token + user response.
  • src/main/java/com/smartoffice/auth/api/UserResponse.java: user metadata response.
  • src/main/java/com/smartoffice/auth/api/MessageResponse.java: simple message payload.

Auth Domain and Persistence

  • src/main/java/com/smartoffice/auth/domain/UserAccount.java: user entity mapped to users table.
  • src/main/java/com/smartoffice/auth/domain/Role.java: role enum (EMPLOYEE, MANAGER, ADMIN).
  • src/main/java/com/smartoffice/auth/repository/UserAccountRepository.java: JPA repository methods for user lookup and duplication checks.

Auth and Security Services

  • src/main/java/com/smartoffice/auth/service/AuthService.java: core auth business logic.
  • src/main/java/com/smartoffice/auth/service/JwtTokenService.java: JWT issue/parse/type checks/revocation integration.
  • src/main/java/com/smartoffice/auth/service/LoginAttemptService.java: lockout algorithm.
  • src/main/java/com/smartoffice/auth/service/CustomUserDetailsService.java: user lookup for Spring Security auth provider.

Security State Store Abstractions

  • src/main/java/com/smartoffice/auth/service/store/RefreshTokenRevocationStore.java: revocation interface.
  • src/main/java/com/smartoffice/auth/service/store/InMemoryRefreshTokenRevocationStore.java: local revocation with expiry cleanup.
  • src/main/java/com/smartoffice/auth/service/store/RedisRefreshTokenRevocationStore.java: Redis revocation with TTL keys.
  • src/main/java/com/smartoffice/auth/service/store/LoginAttemptStore.java: login-attempt interface.
  • src/main/java/com/smartoffice/auth/service/store/LoginAttemptSnapshot.java: immutable lockout state.
  • src/main/java/com/smartoffice/auth/service/store/InMemoryLoginAttemptStore.java: local map store with TTL cleanup.
  • src/main/java/com/smartoffice/auth/service/store/RedisLoginAttemptStore.java: serialized snapshot state in Redis.

HTTP Security and Protected Endpoints

  • src/main/java/com/smartoffice/common/security/SecurityConfig.java: stateless config and route permits.
  • src/main/java/com/smartoffice/common/security/JwtAuthenticationFilter.java: bearer token parsing and context setup.
  • src/main/java/com/smartoffice/security/api/SecureAccessController.java: role-gated sample APIs.

Exception Handling

  • src/main/java/com/smartoffice/common/exception/ApiExceptionHandler.java: central exception-to-response mapping.
  • src/main/java/com/smartoffice/common/exception/TooManyRequestsException.java: lockout-specific exception type.
