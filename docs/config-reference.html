This page maps configuration keys to behavior so you can understand what changes when values are updated.

Primary Config Files

  • src/main/resources/application.yml: default local runtime
  • src/main/resources/application-local-redis.yml: enables Redis-backed security state

Default Data and DB Setup

  • H2 in-memory DB: jdbc:h2:mem:smartoffice
  • Hibernate DDL mode: update
  • H2 console enabled: /h2-console

Redis Profile Behavior

When the local redis profile config is active:

  • Redis host: 127.0.0.1, port: 6379
  • security.redis.enabled=true
  • State persists outside the JVM process

Run Examples

# default mode (in-memory security state)
mvn spring-boot:run

# redis mode via helper script
./docs/run-local-redis.sh

Production Notes

  • Use a strong jwt.secret from environment variables or secret manager.
  • Disable H2 console in non-development environments.
  • Prefer Redis mode if running multiple instances.
