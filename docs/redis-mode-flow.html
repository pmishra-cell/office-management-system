Purpose

Redis mode enables shared security state across service instances for:

  • Refresh token revocation
  • Login-attempt lockout state

Key Files

  • docs/docker-compose.redis.yml
  • src/main/resources/application-local-redis.yml
  • src/main/java/com/smartoffice/auth/service/store/RedisRefreshTokenRevocationStore.java
  • src/main/java/com/smartoffice/auth/service/store/RedisLoginAttemptStore.java
  • docs/run-local-redis.sh
  • docs/stop-local-redis.sh

Bean Selection Rules

  • Default: security.redis.enabled=false uses in-memory stores.
  • Redis mode: security.redis.enabled=true switches to Redis stores.

How Data Is Stored In Redis

  • Revoked refresh tokens: <key-prefix>:revoked-refresh:<token> with TTL to expiry.
  • Login attempts: <key-prefix>:login-attempts:<identity> serialized snapshot with TTL.

One-Command Local Run

./docs/run-local-redis.sh

Script actions:

  1. Start Redis via Docker Compose
  2. Run Spring Boot with local-redis profile

Stop Command

./docs/stop-local-redis.sh

Recommended Verification

  1. Register/login to get tokens.
  2. Call logout with refresh token.
  3. Call refresh with same token and expect unauthorized response.
If Redis is unavailable while security.redis.enabled=true, startup or runtime store calls will fail. Keep Redis container running in this mode.
