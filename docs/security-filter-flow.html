SecurityConfig Behavior

  • CSRF is disabled for stateless API usage.
  • Session policy is stateless.
  • Public routes are permitted: auth endpoints, health/info, H2 console, swagger docs.
  • JwtAuthenticationFilter runs before username-password filter.

JwtAuthenticationFilter Steps

  1. Read Authorization header.
  2. Require Bearer <token> format.
  3. Parse token claims via JwtTokenService.parseClaims.
  4. Only accept tokens with type=access.
  5. Create authentication using email + ROLE_* authority.
  6. Put auth object into SecurityContext.
  7. Continue filter chain.

Role Authorization

  • /api/v1/secure/employee: EMPLOYEE, MANAGER, ADMIN
  • /api/v1/secure/manager: MANAGER, ADMIN
  • /api/v1/secure/admin: ADMIN only

What Happens On Invalid Tokens

  • Filter ignores invalid tokens and continues.
  • Protected endpoint access then fails through normal security checks.
