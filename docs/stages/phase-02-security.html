Planned Scope

  • Refresh token endpoint
  • Logout and blacklist strategy
  • RBAC and method-level authorization checks
  • Rate limiting and brute-force defenses

Implemented in This Phase

  • POST /api/v1/auth/refresh rotates refresh token and returns a new access/refresh pair
  • POST /api/v1/auth/logout revokes refresh token (in-memory blacklist)
  • Refresh validation enforces token type and blocks revoked tokens
  • JWT authentication filter enables Bearer-token based authorization on protected APIs
  • Role-based sample endpoints added with @PreAuthorize checks for EMPLOYEE, MANAGER, and ADMIN roles
  • Brute-force defense added: repeated failed login attempts trigger temporary account lockout with HTTP 429
  • Optional Redis-backed shared security state added for refresh token revocation and login-attempt lockout across multiple instances

Enable Redis-Backed Security State

  • Set security.redis.enabled: true in configuration
  • Configure Redis connection under Spring defaults (for example spring.data.redis.host and spring.data.redis.port)
  • Optional: customize key namespace with security.redis.key-prefix

Local One-Command Validation

  • Start Redis + app with local profile: ./docs/run-local-redis.sh
  • Profile file: src/main/resources/application-local-redis.yml
  • Compose file: docker-compose.redis.yml
  • Stop Redis service: ./docs/stop-local-redis.sh
